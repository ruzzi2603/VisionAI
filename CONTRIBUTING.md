# Contribuindo para VisionGuard AI

Obrigado por querer contribuir! Este documento descreve o fluxo de trabalho recomendado para bugs, melhorias e novos recursos.

## Como começar
1. Faça um fork do repositório.
2. Clone seu fork localmente.
3. Crie uma branch descritiva:
   - `feature/adicionar-camera-local`
   - `fix/evento-ingest-erro`
4. Faça commit com mensagens claras e objetivas.

## Padrões de commit
- Use frases no imperativo: `Corrige`, `Adiciona`, `Melhora`.
- Relacione o commit ao problema ou feature.
- Evite commits muito grandes; prefira mudanças pequenas e iterativas.

## Qualidade do código
- Mantenha o estilo Python consistente com PEP 8.
- No frontend, siga convenções de JavaScript/React e Tailwind.
- Escreva código claro e com responsabilidade.

## Testes e revisão
- Execute as migrações necessárias antes de testar.
- Verifique se o backend e frontend carregam corretamente.
- Se possível, inclua exemplos de uso ou instruções para validar a mudança.

## Fluxo de Pull Request
1. Atualize sua branch a partir da `main`.
2. Abra o PR no repositório original.
3. Descreva o problema, a solução e os passos de validação.
4. Aguarde revisão e responda a feedbacks.

## Como reportar bugs
- Abra uma issue com o título claro.
- Inclua passos para reproduzir, comportamento esperado e observado.
- Anexe logs ou capturas de tela se existir.

## Solicitações de recursos
- Explique o caso de uso.
- Indique se há dependências ou requisitos específicos.
- Seja objetivo sobre o valor para o projeto.
